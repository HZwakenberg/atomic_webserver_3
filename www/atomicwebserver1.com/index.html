What is Atomic Webserver 3

Atomic Web Server 3, is a multi threaded version of the origonal Atomic Web Server with minimal locks, featuring a PB Tag Preprocessor, URI Handlers, Reverse Proxy and an optional Virtual File System to serve assets from memory. With Atomic Web Server 3 you can create a single exe with no dependencies to Host web gui applications or Spider Basic applications for the desktop and serve, locally to a web gadget or a web browser. Uri handlers let you create dynamic content and respond to POST or GET requests. The Tag Preprocessor fills in elements within html using runtime procedures and eliminates the need for CGI or interpreted languages like PHP, Python, Lua or Node JS for server side processing unleashing the pure power of PB on the web. One exe with no dependencies when using the optional virtual file system.

What does it do and what do you want?

Atomic web server can be used as a simple test server localhost or across the Lan or Wan, you don't need CGI or a PHP interpreter for server side processing and you can embed all your assets into a datasection with a virtual file system that's equivelnt to your development www folder, so once your ready to deploy you just create a pack with one call and pack the www folder, include the generated binary into a data section and recompile your source to produce an exe with no dependendencies.

Full stack web development with PB

No need for large foot print server stacks or server side scripting languages php, node js, Lua or Python. You have all the power and features of PB available to you

Create cloud services, Intranet applications, Desktop applications

The built in PB pre processor delivers an ASP or PHP like experience simply tag your html saved as a ".pbh" file, with the tag "< ?PB SomeFunction()/>" declare the prodcure as runtime SomeFunction and it'll inline the output.
